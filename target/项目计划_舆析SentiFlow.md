# 舆析 SentiFlow 项目计划

## 1. 项目概述

### 1.1 项目名称
**舆析 SentiFlow：面向企业舆情与评论批量分析的智能平台**

### 1.2 项目定位
本项目聚焦企业在舆情监测、评论分析、用户反馈归因与风险预警中的实际需求，建设一套支持**批量文本接入、智能分析、异步处理、结果回传与可视化报表**的 AI 分析平台。

平台面向品牌、公关、运营、客服与产品团队，帮助其从海量评论、社媒文本、工单记录和用户反馈中，快速识别情绪趋势、热点问题、负面风险与改进方向。

### 1.3 项目目标
- 建立面向评论与舆情文本的批量采集与处理能力。
- 提供情感分析、主题识别、关键词提取、问题归因等核心能力。
- 建立异步任务执行与状态追踪机制，支撑中大规模文本分析任务。
- 输出可视化结果与统计报表，辅助企业进行运营决策与风险响应。
- 为后续扩展多平台接入、批量评测和预警通知能力预留架构空间。

### 1.4 目标用户
- **品牌与公关团队**：监测品牌口碑、负面舆情和风险扩散趋势。
- **运营团队**：分析用户评论、识别高频反馈问题与内容热点。
- **客服团队**：从工单与投诉文本中提炼核心诉求与改进方向。
- **产品团队**：基于用户声音发现功能缺陷、需求倾向与体验问题。

### 1.5 典型应用场景
- 电商平台商品评论批量分析
- 社交媒体品牌舆情监测与预警
- 客服工单/投诉文本归因分析
- App 商店评论情绪趋势分析
- 不同 Prompt 或分析策略的批量效果对比

## 2. 项目范围

### 2.1 第一阶段（MVP）范围
本阶段聚焦“跑通舆情分析闭环”，交付以下核心能力：
- 批量导入评论/舆情文本数据（CSV/JSON）
- 文本预处理与任务创建
- 情感分析、关键词提取、主题归类
- 高频问题识别与负面样本筛选
- 异步任务执行与状态追踪
- 分析结果展示与基础报表导出
- 执行日志查看与基础失败重试

### 2.2 暂不纳入 MVP 的能力
以下内容作为第二阶段及后续增强方向：
- 多平台实时采集与自动爬取
- Redis Cluster 集群化部署
- Spark 大规模离线仿真与并行计算
- 复杂权限系统与多租户能力
- 企业微信/邮件/短信预警通知
- 高级报表模板与自动周报生成

### 2.3 关键交付物
- 评论/舆情数据导入模块
- 智能分析任务流与后端服务
- 情感/主题/关键词分析模块
- 结果展示界面与基础可视化报表
- 项目文档、测试文档与部署说明

## 3. 功能设计

### 3.1 数据接入模块
- 支持导入商品评论、社媒文本、客服记录等批量文本数据
- 支持 CSV、JSON 等基础格式
- 支持为任务附加来源平台、时间范围、产品线等元信息
- 支持任务创建、任务队列投递与状态记录

### 3.2 智能分析模块
- 情感分类：正向 / 中性 / 负向
- 关键词提取：高频词、热点词、风险词
- 主题识别：对评论内容进行主题聚类或归类
- 问题归因：识别物流、质量、价格、服务、功能等问题类别
- 样本摘要：提取代表性评论，辅助人工快速理解问题

### 3.3 风险预警模块
- 筛选负面评论与高风险文本
- 识别负面主题集中爆发趋势
- 标记敏感词、投诉升级倾向和异常波动
- 为后续接入自动通知能力预留接口

### 3.4 报表与展示模块
- 展示情感分布、主题占比、关键词排行
- 展示负面评论趋势和高频问题排行
- 提供典型评论样本列表
- 支持导出基础分析结果与报表数据

## 4. 技术方案

### 4.1 技术栈
**前端**
- React
- TypeScript
- TailwindCSS
- ECharts 或 Recharts（基础图表展示）

**后端**
- Python 3.9+
- FastAPI
- Pydantic

**中间件与存储**
- RabbitMQ（异步任务调度）
- Redis（任务状态、缓存、上下文管理）
- PostgreSQL（任务元数据、分析结果、配置数据持久化）

**AI 与执行能力**
- LLM API（情感分析、摘要、归因辅助）
- Python 文本处理逻辑（清洗、统计、规则抽取）
- Docker（用于隔离执行可扩展分析任务）

**部署与监控**
- Docker Compose
- Prometheus
- Grafana

### 4.2 架构原则
- **场景优先**：围绕舆情/评论分析场景设计功能，不做泛化过度的平台化扩张。
- **先闭环、后扩展**：优先实现数据导入、分析、展示和导出的最小可用闭环。
- **异步优先**：面向批量任务设计执行链路，避免长任务阻塞系统响应。
- **可观测优先**：每个任务具备状态、日志、耗时与错误信息可追踪能力。
- **可演进性**：为后续多平台采集、预警通知、批量评测与离线并行计算预留接口。

## 8. 风险分析与应对策略

| 风险项 | 风险说明 | 应对策略 |
|---|---|---|
| 数据来源差异大 | 不同平台评论格式不统一，清洗成本高 | 首版聚焦统一格式导入，逐步扩展数据适配器 |
| 模型分析不稳定 | 情感分析和归因结果可能波动 | 结合规则与模型双重策略，并保留人工校验样本 |
| 任务量增长带来性能压力 | 批量文本分析容易造成队列积压 | 使用异步队列削峰，按任务批次处理 |
| 结果可解释性不足 | 用户难以理解模型为何给出某结论 | 输出关键词、代表样本和分类依据摘要 |
| 范围蔓延 | 容易从评论分析扩展成泛化平台 | 严格围绕舆情/评论分析场景推进迭代 |

## 9. 验收指标

### 9.1 功能指标
- 支持 CSV/JSON 评论数据导入
- 支持情感分析、关键词提取、主题归类与问题归因
- 支持任务状态查看、执行日志查看与失败重试
- 支持结果展示与基础报表导出

### 9.2 性能指标
- 单次支持处理至少 1 万条基础评论数据（可分批执行）
- 支持多任务顺序进入队列执行
- 关键页面与任务查询接口满足演示和测试要求

### 9.3 工程指标
- 输出完整部署文档与使用说明
- 完成核心模块测试
- 具备基础监控和日志追踪能力

### 9.4 业务效果指标
- 可识别主要情感倾向与高频问题类别
- 可筛选代表性负面样本
- 可辅助生成可读的分析结论与改进建议

## 10. 后续演进方向

在 MVP 验证通过后，项目可向以下方向扩展：
- 接入微博、小红书、抖音、App 商店等多平台数据源
- 增加风险预警通知能力（企业微信、邮件等）
- 引入更强的主题聚类与趋势检测能力
- 建设 Prompt/策略批量评测模块，对比不同分析方案效果
- 引入 Redis Cluster 与 Spark，支持更大规模并发和离线分析
- 增加自动周报、竞品对比和品牌画像等高级能力
